# Proofroom implementation mapping

This document maps the approved **Proofroom — The Evidence Ledger** design to the validated PolicyProof application. The design package controls presentation. Existing application schemas, deterministic checks, GPT-5.6 routes, source validation, reviewer-decision behavior, and security limits remain authoritative.

## Implementation constraints

- Keep the existing single React state owner in `DemoReviewWorkspace`; do not add Zustand or another runtime dependency.
- Keep stable domain identifiers such as `CTRL-APPROVAL` and `DOC-PO-1042`. Display aliases such as `R-01` and `DEC-01` are derived presentation references, never replacement IDs.
- Keep `.txt`, `.md`, and `.json`, 1 MB per file, and 10-file limits. The prototype's PDF/DOCX/XLSX/20 MB copy is intentionally not implemented because it conflicts with validated product scope.
- Keep reviewer decisions resetting when a review is rerun. This is the existing, tested application behavior and the integration brief explicitly requires it.
- Keep exact policy requirements, document names, evidence excerpts, comments, identifiers, and source locators untranslated.
- Use the approved font family names with robust local fallbacks. Do not add remote font requests or a font package during this no-install phase.
- Use application data to render outcomes, evidence counts, the EUR/USD comparison, threshold changes, and receipts. Do not copy static prototype results.
- Temporary screenshots remain under ignored `test-results/proofroom-integration/`.

3. Playwright covers the complete judge path at 1440 px, French state preservation, 390 px Review behavior, keyboard navigation, mocked provider failure, threshold FAIL→PASS, and receipt actions without a provider request.
